# 项目说明

这是 Maeve 用于 Ai-Thinker 海外社媒运营工作的个人工具集。

项目主要服务于 LinkedIn、Facebook、TikTok、YouTube 等平台的内容创作、营销物料整理和数据辅助分析。

重点内容形式包括：
- 9:16 竖版图文海报
- 9:16 竖版短视频
- 产品推广页面
- 内容发布辅助工具
- 社媒数据整理和分析工具

# 技术约定

- 优先使用原生 HTML、CSS 和 JavaScript。
- 不使用 Vue、React、jQuery 等前端框架，除非明确要求。
- 简单工具优先写在单个 HTML 文件里。
- 文件命名使用小写英文和短横线，例如 `todo-list.html`、`poster-planner.html`。
- 代码要清晰，适合初学者阅读。
- 关键逻辑可以写中文注释，但不要为了注释而注释。

# 视觉风格

- 整体风格偏现代、简洁、专业、有科技感。
- 优先使用黑、白、灰，以及少量品牌色或高亮色。
- 页面要有足够留白，避免拥挤。
- 字体优先使用 `PingFang SC`、`system-ui`、`sans-serif`。
- UI 要适合移动端查看，尤其要考虑 9:16 竖屏内容。

# 营销内容偏好

- 文案要简短、直接、面向海外用户。
- 英文表达优先自然、清楚，不要堆砌夸张营销词。
- 内容要突出产品卖点、应用场景和用户价值。
- 面向 B2B、开发者、工程师、采购或物联网行业用户时，语气要专业可信。

# 产品与素材要求

- 涉及 Ai-Thinker 的无线通信模块、ESP32、LoRa、UWB、雷达传感器等硬件产品时，要保持真实产品结构。
- 不要把硬件画成夸张科幻风、变形结构或不真实的电路形态。
- 如果需要产品图、海报、视频脚本或展示页，应优先保证信息准确和商业可用。

# 页面与工具偏好

- 不要默认添加冗余 Footer、备案号、无用联系方式。
- 不要把简单工具做复杂。
- 不要引入复杂构建工具。
- 不要随意删除已有文件。
